## Wichtigstes Wissen von S16

### Kritischste Korrektur (für alle künftigen Sessions)
**ConversationEngine ist KEIN Intent-Switch.** Sie ist eine agentische Tool-Use-Schleife:
- Claude entscheidet via `tool_use`
- `AssistantToolRegistry.run(name:inputJSON:)` führt gewhitelistete Tools aus
- Ergebnisse als `tool_result` zurück → Schleife bis `end_turn` / `maxToolRounds=6`
- **S18 + S19 müssen neue `ClaudeToolDefinition` + `run`-Handler in der Registry bauen** — kein "Intent-Case"
- `ConversationEngine.activityLabel` hat hardcodierten switch über Tool-Namen → neue Tools dort ergänzen

### Vier S18-Fallstricke (Kalkulations-Chat-Tool)
1. `schaetze(projektID:freitext:)` hat `EstimateRequestParser.parse(freitext)` intern → Freitext direkt durchreichen, kein Doppel-Parsing
2. `schaetze` SCHREIBT `learningStore.saveSession` → ist OK und gewollt (Referenz für recordAdjustment)
3. **projektID-Lücke**: Tools bekommen nur `inputJSON`, keinen App-Kontext → `scope` (`.project(String)`) aus `send()` in Registry-Aufruf einspeisen
4. `BaselineAnchorProvider` ist Pflicht — StubAnchorProvider gibt mitteNetto == 0

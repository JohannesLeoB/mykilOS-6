## Wichtigstes Wissen von S15

### Cold-Start-Test-Warnung
`schaetze` persistiert seit Schritt 7 jede Session. Tests müssen **frische temp-Dirs** nutzen, damit die Kandidaten-Schwelle (≥3 ähnliche Adjustments) deterministisch bleibt. Sonst verschieben Alt-Sessions aus früheren Läufen die Schwelle.

### BaselineAnchorProvider ist Pflicht
Mit dem leeren `StubAnchorProvider` ist `mitteNetto == 0` — Kalibrierung verschiebt sich nie. `AppState` nutzt korrekt `BaselineAnchorProvider`. Tests müssen ihn auch nutzen.

### Swift-Protokoll-Gotcha
Swift-Protokoll-Anforderungen können keine Default-Argumente haben. S15 löste das via Protokoll-Extension als Convenience-Methode für Rückwärtskompatibilität mit Schritt-7-Tests.
